### 1. 完整 Feature 开发 → 规范化流程 + 自动化测试

对于核心玩法、系统模块（如背包、技能树、网络同步），建议：

- 先写 spec
    
- 用 spec-workflow-mcp 生成需求、设计、任务文档
    
- 用 Claude Code 和 MCP 开发实现
    
- 生成自动化测试（AI 可生成单元测试）
    

虽然前期慢，但**成功率 >90%**，后期维护成本极低。

### 2. 小功能 / 重构 / Bug 修复 → 计划&行动模式（Plan & Act）

“给角色加个受伤音效”、“重构一下摄像机管理器”、“修复背包物品显示错位”——这类需求，直接打开 Claude Code 的“Plan & Act”模式：

- 描述需求
    
- AI 生成修改计划（哪几个文件、改哪几行、加什么函数）
    
- 一键应用（或分步确认）
    
- 手动测试验证
    

效率提升**3~5倍**，尤其适合敏捷开发和快速迭代。

### 3. 代码片段修改 → Copilot 沉浸式编程

当你已经打开文件，只是想“重命名这个变量”、“加个空值判断”、“补全这个 for 循环”——这时候，**Copilot 的即时补全**反而更高效。Claude Code 适合“做菜”，Copilot 适合“加盐”。

> 🎯 **终极组合技**：
> 
> - Rider（导航 + 编译 + MCP）
>     
> - Claude Code（功能开发 + 重构 + 问答）
>     
> - Copilot（片段补全 + 变量命名）
>     
> - spec-workflow-mcp（规范驱动）
>     

### 4. 提前生成常用批处理 & 自动化工具 —— 让 AI 为你打工，而不是你为 AI 收尾

在 UE5 项目中，除了写功能代码，我们往往还要花大量时间在“工程化任务”上：

- 批量格式化代码（clang-format）
    
- 执行自动化测试（Automation System）
    
- 重新生成项目文件（GenerateProjectFiles.bat）
    
- 打包构建（UAT BuildCookRun）
    

这些工作，AI 不能替你“一键完成”，但可以提前帮你生成完整的脚本和工具链！

> ✅ 推荐做法：  
> 🔹 让 Claude Code 为你生成批处理脚本（.bat / .ps1 / Python）
