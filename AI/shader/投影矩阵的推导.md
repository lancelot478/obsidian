# 投影矩阵是怎么得到的

> 日期：2026-03-31

---

## 一、投影矩阵的目的

观察空间是一个**梯形视锥体（Frustum）**，GPU 不好直接裁剪和光栅化它。  
投影矩阵的任务就是把这个不规则的视锥体**压缩变形**成一个规则的立方体，方便后续处理。

```
观察空间（梯形视锥）        裁剪空间（立方体）

      /|                      |‾‾‾‾‾|
     / |                      |     |
    /  | far                  |     |
   /   |           →          |     |
  /    |                      |     |
 / near|                      |_____|
摄像机
```

## 二、透视投影矩阵推导（最常用）

### 前置参数

| 参数 | 含义 |
|------|------|
| `fov` | 垂直视野角（Field of View），如 60° |
| `aspect` | 宽高比（width / height） |
| `near` | 近裁剪面距离 |
| `far` | 远裁剪面距离 |

### 推导过程

**第一步：理解透视投影的本质——相似三角形**

在观察空间，一个点 `(x, y, z)` 投影到近裁剪面上的位置：

```
         y                  y_proj
         |                  |
         |   \              |   \
         P = (x,y,z)         →    P' = (x',y')
             \                       \
              \                        \
               摄像机(0,0,0)            近裁剪面(z=-near)
```

由**相似三角形**：

```
y' / near = y / |z|

y' = near × y / |z|
x' = near × x / |z|
```

> 注意：Unity/OpenGL 观察空间 z 轴朝摄像机后方，物体在摄像机前方时 z 为负值，所以分母用 -z

$$x' = \frac{near \cdot x}{-z}, \quad y' = \frac{near \cdot y}{-z}$$

**第二步：把投影结果归一化到 [-1, 1]**

近裁剪面的范围：
```
垂直半高：t = near × tan(fov/2)
水平半宽：r = t × aspect
```

归一化：

$$x_{ndc} = \frac{x'}{r} = \frac{near \cdot x}{r \cdot (-z)} = \frac{x}{aspect \cdot \tan(fov/2) \cdot (-z)}$$

$$y_{ndc} = \frac{y'}{t} = \frac{near \cdot y}{t \cdot (-z)} = \frac{y}{\tan(fov/2) \cdot (-z)}$$

令 $f = \frac{1}{\tan(fov/2)}$（cotangent，常见缩写），则：

$$x_{ndc} = \frac{f/aspect}{-z} \cdot x, \quad y_{ndc} = \frac{f}{-z} \cdot y$$

**第三步：构造齐次矩阵（保留 -z 用于透视除法）**

GPU 自动做透视除法（÷W），所以矩阵只需让 W 输出 = -z，再把系数填进去：

$$\begin{bmatrix} x_{clip} \\ y_{clip} \\ z_{clip} \\ w_{clip} \end{bmatrix} = M_{proj} \begin{bmatrix} x \\ y \\ z \\ 1 \end{bmatrix}$$

透视除法后：$x_{ndc} = x_{clip}/w_{clip}$，要使 $x_{ndc} = \frac{f/aspect}{-z} \cdot x$

令 $w_{clip} = -z$，则 $x_{clip} = \frac{f}{aspect} \cdot x$

**第四步：处理深度 z（映射到 [-1, 1]）**

z 的映射需要满足：
- $z = -near$ 时，$z_{ndc} = -1$
- $z = -far$ 时，$z_{ndc} = +1$

设 $z_{clip} = Az + B$，则 $z_{ndc} = z_{clip}/w_{clip} = (Az+B)/(-z)$

联立方程：
$$\frac{A(-near)+B}{-(-near)} = -1 \Rightarrow -A \cdot near + B = -near$$
$$\frac{A(-far)+B}{-(-far)} = +1 \Rightarrow -A \cdot far + B = far$$

解方程组：

$$A = -\frac{far+near}{far-near}, \quad B = -\frac{2 \cdot far \cdot near}{far-near}$$

**最终透视投影矩阵（OpenGL/Unity 约定）：**

$$M_{proj} = \begin{bmatrix} \frac{f}{aspect} & 0 & 0 & 0 \\ 0 & f & 0 & 0 \\ 0 & 0 & -\frac{far+near}{far-near} & -\frac{2 \cdot far \cdot near}{far-near} \\ 0 & 0 & -1 & 0 \end{bmatrix}$$

其中 $f = \cot(fov/2) = \frac{1}{\tan(fov/2)}$

> **第四行 (0, 0, -1, 0)** 是关键：它让 $w_{clip} = -z$，GPU 透视除法后就自动还原出正确的 NDC 坐标

## 三、数值示例

参数：`fov=60°`，`aspect=16/9≈1.778`，`near=0.3`，`far=1000`

```
f = 1/tan(30°) = 1/0.577 ≈ 1.732

f/aspect = 1.732/1.778 ≈ 0.974

A = -(1000+0.3)/(1000-0.3) ≈ -1.0006
B = -(2×1000×0.3)/(1000-0.3) ≈ -0.6002
```

矩阵数值：

```
┌ 0.974   0       0        0      ┐
│ 0       1.732   0        0      │
│ 0       0      -1.0006  -0.6002 │
└ 0       0      -1        0      ┘
```

验证一个点 P=(1, 1, -0.3)（正好在近裁剪面上）：

```
x_clip = 0.974 × 1 = 0.974
y_clip = 1.732 × 1 = 1.732
z_clip = -1.0006×(-0.3) + (-0.6002) = 0.3002 - 0.6002 = -0.3
w_clip = -(-0.3) = 0.3

x_ndc = 0.974/0.3 ≈ 3.25  ← 超过屏幕宽度被裁剪，正确
z_ndc = -0.3/0.3 = -1      ← 正好在近平面，正确 ✅
```

## 四、正交投影矩阵（对比）

正交投影**没有透视缩小**（远近物体一样大），常用于 UI、2D游戏、工程视图。

推导更简单，直接线性映射：

$$M_{ortho} = \begin{bmatrix} \frac{2}{r-l} & 0 & 0 & -\frac{r+l}{r-l} \\ 0 & \frac{2}{t-b} & 0 & -\frac{t+b}{t-b} \\ 0 & 0 & -\frac{2}{far-near} & -\frac{far+near}{far-near} \\ 0 & 0 & 0 & 1 \end{bmatrix}$$

> 注意最后一行是 `(0,0,0,1)` 而非 `(0,0,-1,0)`，W 恒为 1，**没有透视除法效果**

## 五、两种投影对比

```
透视投影（Perspective）          正交投影（Orthographic）

     /|                           |‾‾‾‾‾‾‾|
    / |                           |       |
   /  |  远处小                   |       |  远近一样大
  /   |                           |       |
 / near|                          |_______|

W = -z（不为1）                   W = 1（恒定）
有透视除法                         无透视除法
视锥是梯形                         视锥是长方体
```

| 特性 | 透视投影 | 正交投影 |
|------|----------|----------|
| 近大远小 | ✅ | ❌ |
| W 分量 | = -z（变化） | = 1（固定） |
| 适用场景 | 3D 游戏、第一/第三人称 | UI、2D游戏、工程图 |
| depth 线性 | ❌ 非线性 | ✅ 线性 |

## 六、在 Shader 中如何用

```hlsl
// Unity 中投影矩阵已由引擎自动设置
// 顶点着色器中直接用即可

// 方式一：一步到位（模型空间 → 裁剪空间）
o.pos = UnityObjectToClipPos(v.vertex);
// 等价于：mul(UNITY_MATRIX_MVP, v.vertex)

// 方式二：分步（有时候需要中间的观察空间数据）
float4 viewPos = mul(UNITY_MATRIX_MV, v.vertex);  // 模型 → 观察
// ... 在观察空间做一些事（如描边膨胀）...
o.pos = mul(UNITY_MATRIX_P, viewPos);              // 观察 → 裁剪

// 获取投影矩阵（用于自定义计算）
float4x4 proj = UNITY_MATRIX_P;
// 访问 fov 相关数据：proj[1][1] = f = cot(fov/2)
float fov_rad = 2.0 * atan(1.0 / proj[1][1]);
```

## 七、为什么深度是非线性的？

透视投影后，深度 $z_{ndc}$ 与真实距离 $z$ 的关系：

$$z_{ndc} = \frac{-\frac{far+near}{far-near} \cdot z - \frac{2 \cdot far \cdot near}{far-near}}{-z}$$

这是一个关于 z 的双曲线，**靠近 near 处精度高，靠近 far 处精度急剧下降**：

```
z_ndc
 1.0 |                                    ●  far
     |                               ●
     |                          ●
     |                    ●
     |              ●
     |        ●
-1.0 | ●  near
     +------------------------------------------→ 真实深度 z
      near                                    far

近处精度密集，远处精度稀疏 → 远处容易 Z-Fighting！
```

这就是为什么远处物体容易出现 **Z-Fighting**，解决方法包括：
- 调大 near 值（near 越大，精度分布越均匀）
- 使用 **Reversed-Z**（将深度范围翻转为 1→0，精度更均匀）
- 使用 **对数深度缓冲（Logarithmic Depth Buffer）**
