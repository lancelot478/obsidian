#Muffin

格数据目前采用的是Json的形式，在初始化的时候解析到Lua虚拟机内存

也就是 Excel元数据 → Json → Lua

优化方案是去除中间环节，Excel → Lua

## 效率对比

这里以比较大的两张表：Skill、Item表做测试

|       |       |            |           |
| ----- | ----- | ---------- | --------- |
| 方式    | 表格    | 解析时间（秒）    | 内存分配（Kb）  |
| Json  | Item  | 0.049      | 324       |
| Skill | 0.28  | 1651       |           |
| Lua   | Item  | 0.002      | 177(↓45%) |
| Skill | 0.006 | 1137（↓31%） |           |

## 进阶优化

- 配置结构优化：Lua的Table有两种形式，可以把数据转成ArrayTable的形式，建立索引访问数据，内存可以小很多
- 清楚冗余数据：配置表内存在大量的0、空表、空串等，可以省略掉，在接口层使用常量值代替
- JSON不支持数值作为key，所以现在访问配置数据需要把数字转为字符串，Lua没有这个限制

## 好处

- 解析效率提升
- 内存减小
- 方便管理（动态加载、卸载支持）
