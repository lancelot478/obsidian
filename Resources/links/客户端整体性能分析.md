# 资源

资源手动标记，当被引用到的资源没有被标记，被引用资源会被打进引用该资源所在的Bundle中。

当一份资源被多个Bundle内的资源引用后，这份资源也会同时存在于多个Bundle中，造成资源的冗余

## 冗余情况

|类别|总量|冗余|冗余比例|
|---|---|---|---|
|资源|6749|616|9%|
|AssetBundle|423|376|89%|

通过资源冗余比例和Bundle冗余比例可以看出，有部分(9%)的资源被大量的资源依赖，但是这部分资源并没有打独立的Bundle包

导致这部分资源冗余了

*** 可以通过 Window - AssetBundle Browser界面查看Bundle资源的信息**

### 冗余导致的内存问题

不完全统计，仅Texture2D和Sprite两种类型的资源，冗余的Native内存达40M

# CPU

|   |   |   |   |   |
|---|---|---|---|---|
|类目|描述|   |   |影响程度|
|跨域调用|Lua调用C#的接口还是比较昂贵的，单次零点几毫秒，每一帧处理的内容多的情况下，脚本耗时严重  <br>建议养成意识，避免多次重复跨域调用|   |   |影响大|
|PlayerPrefs|每隔几秒左右会有一个上百次的PlayerPref的GetInt、SetInt调用，掉帧十分明显。建议优化|   |   |影响很大，过一会掉一次帧就是这个原因|
|Active/Deactive|每次调用都会遍历整个Node结构，并且会触发一系列Mono脚本的生命周期函数，部分Object建议移出摄像机不渲染|   |   |影响一般|

## GC部分

目前占比比较多的最集中的部分，在于协程的Yield对象创建

创建Yield对象，如果是固定参数固定类型的对象，尽量一次创建，多次使用

如

**.Net Code**

|   |   |
|---|---|
|1<br><br>2<br><br>3<br><br>4<br><br>5<br><br>6<br><br>7|`// 原代码`<br><br>`yield` `return` `new` `WaitForEndOfFrame()`<br><br>`// 替换`<br><br>`// waitForEndOfFrame可多次使用`<br><br>`var waitForEndOfFrame =` `new` `WaitForEndOfFrame()`<br><br>`yield` `return` `waitForEndOfFrame`|
